전체 슬라이드 문구를 아래 기준으로 다시 다듬어줘.

- 한 슬라이드에는 하나의 핵심 메시지만
- 제목은 짧고 명확하게
- 본문은 1~2줄 중심
- 카드는 3~4개 이하로
- 체크리스트는 6개 이하로
- 프롬프트 전문은 슬라이드에 길게 넣지 않기
- 설명문은 강사가 말로 풀 수 있으니 슬라이드에는 핵심만 남기기
- 초급자가 바로 이해할 수 있는 말 사용
- 개발자 중심 표현은 쉽게 풀어쓰기
- 역할 부여는 필수 공식처럼 보이지 않게 하기
- “기획 → 설계와 문서화 → 구현 → 검증” 용어를 일관되게 사용
- “요청 → 확인 → 다시 요청” 흐름을 전체적으로 유지

문구 예시:

- `AI는 마법 지팡이가 아니라 작업 파트너`
- `좋은 질문보다 중요한 것은 결과를 보고 다시 묻는 힘`
- `역할은 필수가 아니라 관점 조정`
- `작게 만들수록 실습은 선명해진다`
- `AI가 만들었다고 끝난 것이 아니다`
- `에러는 실패가 아니라 다시 물어볼 재료`
