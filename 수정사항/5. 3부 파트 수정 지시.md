3부는 8장 정도로 압축해줘.  
3부는 이 강의의 핵심 실습 파트이므로 너무 많이 줄이기보다, 실습 중 꼭 봐야 하는 슬라이드만 남겨줘.

현재 3부는 기획, 문서화, SQL 요청, Supabase 확인, 구현 요청, 구현 답변 확인, 검증, 테스트 문장, 검증 체크리스트 요청, 에러 대처 등으로 너무 세세하게 나뉘어 있어.  
이 중 긴 프롬프트 전문과 상세 설명은 별도 `실습 프롬프트 모음`으로 빼고, 슬라이드는 흐름 중심으로 압축해줘.

## 3부 1장: 바이브코딩 맛보기

추천 제목:  
`바이브코딩은 요청하고 확인하며 작은 결과물을 만드는 경험`

핵심 문장:  
`오늘의 목표는 완성된 서비스가 아니라, AI와 함께 기획, 설계와 문서화, 구현, 검증을 한 번 경험하는 것입니다.`

## 3부 2장: 공통 예제

추천 제목:  
`공통 예제: 오늘의 기록장`

핵심 설명:  
`입력하고, 저장하고, 다시 확인하는 작은 웹페이지`

넣을 것:

- 오늘의 기록 입력
- 저장 버튼
- 저장된 기록 목록
- Supabase DB 저장
- 새로고침 후 다시 불러오기

제외할 것:

- 로그인
- 검색
- 통계
- 관리자 기능
- 복잡한 권한 관리

현재 `공통 예제`와 `기획하기` 슬라이드는 하나로 합쳐도 좋아.

## 3부 3장: 실습 흐름 한 장 요약

추천 제목:  
`오늘의 기록장을 만드는 순서`

4단계:

1. 기획 문서 만들기
2. Supabase 테이블 만들기
3. 웹페이지 구현 요청하기
4. 저장과 불러오기 검증하기

이 슬라이드는 3부 전체의 지도 역할을 하게 해줘.

## 3부 4장: 핵심 프롬프트 1 - 기획 문서

추천 제목:  
`먼저 짧은 기획 문서를 만든다`

슬라이드에는 프롬프트 전문을 다 넣지 말고 요약본만 넣어줘.

예시:  
`오늘의 기록장 웹페이지를 만들기 위한 짧은 기획 문서를 만들어줘. 목적, 입력 항목, 저장 동작, 화면 결과, 검증 체크리스트를 포함해줘.`

하단 문구:  
`자세한 프롬프트는 실습 프롬프트 모음에서 복사합니다.`

## 3부 5장: 핵심 프롬프트 2 - 구현 요청

추천 제목:  
`문서를 기준으로 구현을 요청한다`

요약 프롬프트:  
`이 기획 문서와 Supabase SQL을 기준으로 웹페이지를 만들어줘. 저장, 불러오기, 실행 방법, 전체 코드를 포함해줘.`

강조:  
`기준 문서를 함께 주면 AI의 답변이 더 선명해집니다.`

기존 긴 구현 요청 전문은 슬라이드에서 빼고 실습 프롬프트 모음으로 이동해줘.

## 3부 6장: 검증 체크리스트

추천 제목:  
`AI가 만들었다고 끝난 것이 아니다`

체크리스트:

- 문서와 화면이 같은가?
- 저장 버튼이 동작하는가?
- Supabase DB에 들어갔는가?
- 새로고침 후 다시 보이는가?
- 빈 입력값일 때 안내가 나오는가?
- 화면이 너무 복잡하지 않은가?

이 슬라이드는 꼭 유지해줘.  
초급자에게 가장 중요한 실습 감각이야.

## 3부 7장: 에러 대처

추천 제목:  
`에러는 실패가 아니라 다시 물어볼 재료`

핵심 프롬프트:  
`나는 코딩을 잘 몰라. 원인을 쉽게 설명해주고, 고친 전체 코드를 다시 줘.`

설명:

- 에러 메시지를 그대로 복사한다.
- 현재 상황을 같이 말한다.
- 원하는 결과를 다시 말한다.

## 3부 8장: 내 아이디어로 바꾸기

추천 제목:  
`같은 흐름을 내 아이디어에 적용한다`

4단계:

1. 주제 바꾸기
2. 입력 항목 바꾸기
3. 문서 다시 만들기
4. 다시 구현하기

예시:

- 독서 기록장
- 운동 루틴 기록
- 상품 아이디어 메모장
- 고객 상담 메모장
