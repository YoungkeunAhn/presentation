1

기획 문서 요청 프롬프트

데일리 노트 웹페이지를 만들기 위한 짧은 기획 문서를 만들어줘.

목표:
사용자가 오늘의 기록을 남기고 저장한 기록을 다시 확인할 수 있는 웹페이지를 만들고 싶어.
저장한 기록은 수정하거나 삭제할 수 있어야 하고, 키워드로 검색할 수도 있어야 해.
저장한 내용은 Supabase DB에도 저장되게 할 거야.

문서에 포함할 내용:
- 이 웹페이지의 목적
- 사용자가 입력해야 할 정보
- 저장 버튼을 눌렀을 때의 동작
- 저장된 기록을 수정하고 삭제하는 동작
- 키워드로 기록을 검색하는 동작
- Supabase에 저장되어야 할 데이터
- 화면에서 확인해야 할 결과
- 완성 후 검증 체크리스트

문서를 저장할 때는 docs 폴더 안에 기능별 설명 문서를 따로 나누어 저장해줘.
그리고 전체 내용을 한눈에 볼 수 있는 메인 문서도 함께 만들고, 그 메인 문서에서 각 기능별 설명 문서를 링크로 참조할 수 있게 정리해줘.
파일 이름은 기능명으로 해줘.

2

Supabase SQL 요청 프롬프트

docs 폴더에 저장된 문서를 보고 Supabase SQL Editor에서 실행할 SQL을 만들어줘.

요청:
- 필요한 테이블 이름과 컬럼을 스스로 판단해줘.
- SQL Editor에 바로 붙여넣어 실행할 수 있는 전체 SQL을 줘.
- Row Level Security를 켜고, 초보자 실습용으로 select, insert, update, delete가 가능하도록 policy를 포함해줘.
- 같은 policy가 이미 있어도 다시 실행할 수 있게 drop policy if exists를 포함해줘.
- 설명은 짧게 하고, 실행할 SQL은 docs 폴더 안에 저장해줘.

3

구현 요청 프롬프트

docs에 저장된 문서들을 기준으로 웹페이지를 만들어줘.

요청:
- 사용자가 기록을 입력하고 저장할 수 있게 해줘.
- 저장 버튼을 누르면 Supabase에 저장되게 해줘.
- 저장된 기록을 수정하고 삭제할 수 있게 해줘.
- 키워드로 기록을 검색할 수 있게 해줘.
- 페이지를 처음 열 때 Supabase에서 저장된 기록을 불러와 보여줘.
- Supabase Project URL과 anon public key를 넣는 위치를 표시해줘.
- service_role key는 절대 사용하지 않게 해줘.
- 초보자가 따라 할 수 있게 필요한 파일, 설치 방법, 실행 방법을 순서대로 알려줘.
- 최종 코드는 생략하지 말고 전체를 줘.

4

검증 체크리스트 요청 프롬프트

docs에 저장된 문서와 현재 결과물을 기준으로
초보자가 확인할 수 있는 검증 체크리스트를 만들어줘.

확인하고 싶은 것:
- 화면이 기획 문서와 맞는지
- 저장 버튼이 작동하는지
- Supabase DB에 데이터가 저장되는지
- 저장된 기록을 수정하고 삭제할 수 있는지
- 키워드로 기록을 검색할 수 있는지
- 새로고침 후 기록이 다시 보이는지
- 빈 입력값일 때 안내가 나오는지
- 모바일 화면에서 깨지지 않는지

내 아이디어로 바꾸기 프롬프트

오늘 만든 데일리 노트 흐름을 아래 아이디어에 맞게 바꾸고 싶어.

새 아이디어:
[예: 독서 기록장 / 운동 루틴 기록 / 상품 아이디어 메모장 / 고객 상담 메모장]

바꾸고 싶은 내용:
- 기록할 입력 항목: [2~3개만 적기]
- 저장 후 화면에 보여줄 내용: [목록에 보이면 좋을 내용 적기]
- 제외할 기능: 로그인, 통계, 관리자 기능

요청:
1. 이 아이디어에 맞는 짧은 기획 문서를 다시 만들어줘.
2. 필요한 Supabase 테이블 구조를 제안해줘.
3. 기존 웹페이지 코드에서 바꿔야 할 부분을 초보자 기준으로 설명해줘.
4. 가능하면 고친 전체 코드를 다시 줘.
