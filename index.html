오늘의 흐름

오늘 익힐 AI 활용 흐름

막막함을 줄이고, 요청하고, 작게 만들고, 다시 개선합니다.

1부 · 관점

AI는 마법 지팡이가 아니라 작업 파트너

AI는 대신 완성해주는 존재가 아니라, 함께 정리하고 고쳐가는 도구입니다.

좋은 질문보다 중요한 것은 결과를 보고 다시 묻는 힘입니다.

처음부터 완벽한 답을 기대하지 않습니다.

1부 · 막막함

막막함은 능력 부족이 아니라 낯섦에서 온다

자주 드는 생각

  • 무엇을 물어봐야 할지 모르겠다
  • 내가 너무 몰라서 못 쓸 것 같다
  • 이상한 답이 나오면 어떻게 해야 할지 모르겠다

오늘의 관점

  • 상황을 설명한다
  • 원하는 결과를 말한다
  • 나온 결과를 확인한다
  • 부족한 부분을 다시 요청한다
막막함은 작게 쪼개면 다룰 수 있습니다.

1부 · 도구 역할

도구 이름보다 역할을 먼저 본다

AI 작업자

생각 정리, 요구사항 정리, 코드 작성, 수정 방향 제안을 맡깁니다.

예: Claude, Codex, Antigravity

코드 작업 공간

파일을 보고, 코드를 만들고, 실행 결과를 확인하는 장소입니다.

예: VS Code, Cursor, Antigravity IDE

중요한 질문은 “누구에게 맡기고, 어디서 확인하는가”입니다.

1부 · 개발 흐름

AI 개발은 한 번에 완성품을 받는 일이 아니다

1 기획 무엇을 만들지 작게 정한다.
2 설계와 문서화 화면과 사용자의 행동을 말로 정리한다.
3 구현 AI에게 기준 문서를 주고 결과물을 만들게 한다.
4 검증 내가 원한 결과와 실제 결과를 비교한다.
기획 → 설계와 문서화 → 구현 → 검증

2부 · 프롬프트 핵심

좋은 프롬프트는 멋진 문장이 아니라 분명한 요청

AI에게 잘 맡긴다는 것은 원하는 결과를 알아볼 수 있게 건네는 일입니다.

1 상황 지금 무엇을 하려는지 말합니다.
2 목표 왜 이 결과가 필요한지 말합니다.
3 결과물 무엇을 받아야 하는지 말합니다.
4 조건 범위와 형식을 함께 정합니다.
상황 → 목표 → 결과물 → 조건

오늘의 목표는 완성된 서비스가 아니라, AI와 함께 기획, 설계와 문서화, 구현, 검증을 한 번 경험하는 것입니다.
작게 만들수록 실습은 선명해집니다.

3부 · 공통 예제

공통 예제: 오늘의 기록장

입력하고, 저장하고, 다시 확인하는 작은 웹페이지입니다.

넣을 것

  • 오늘의 기록 입력
  • 저장 버튼
  • 저장된 기록 목록
  • Supabase DB 저장
  • 새로고침 후 다시 불러오기

이번에는 제외

  • 로그인
  • 검색과 통계
  • 관리자 기능
  • 복잡한 권한 관리
작은 예제로 저장과 불러오기를 경험합니다.

3부 · 순서

오늘의 기록장을 만드는 순서

1 기획 문서 목적, 입력, 저장, 화면, 검증 기준을 정리합니다.
2 Supabase 테이블 AI가 만든 SQL을 실행하고 테이블을 확인합니다.
3 웹페이지 구현 기획 문서와 SQL을 기준으로 요청합니다.
4 저장과 불러오기 화면, DB, 새로고침 결과를 확인합니다.
이 슬라이드가 3부의 지도입니다.

3부 · 핵심 프롬프트 1

먼저 짧은 기획 문서를 만든다

오늘의 기록장 웹페이지를 만들기 위한 짧은 기획 문서를 만들어줘.
목적, 입력 항목, 저장 동작, 화면 결과, 검증 체크리스트를 포함해줘.

자세한 프롬프트는 실습 프롬프트 모음에서 복사합니다.

먼저 기준 문서를 만듭니다.

3부 · 핵심 프롬프트 2

문서를 기준으로 구현을 요청한다

이 기획 문서와 Supabase SQL을 기준으로 웹페이지를 만들어줘.
저장, 불러오기, 실행 방법, 전체 코드를 포함해줘.

기준 문서를 함께 주면 AI의 답변이 더 선명해집니다.

긴 구현 요청은 실습 프롬프트 모음으로 분리했습니다.

독서 기록장, 운동 루틴 기록, 상품 아이디어 메모장, 고객 상담 메모장
